# AI 驱动开发方法论

[![支持本书](https://img.shields.io/badge/支持本书-爱发电-946ce6?style=flat-square&logo=afdian&logoColor=white)](https://ifdian.net/a/frully)

本书面向具备产品与开发基础的读者，阐述如何使用 AI 从零驱动一个完整项目的全生命周期：从立项、规划、开发、评审，到测试、发布及持续运维。我们的核心目标是让 AI Agent 成为能够自主执行特性的驱动者，而非仅仅提供代码补全的辅助画笔。

## 开始之前

本书不仅适用于希望将 AI 从“代码补全笔”升级为“研发驱动者”的资深开发者，也同样欢迎产品经理、架构师以及对前沿 AI 协同研发抱有热情的工程探索者。只要你具备基础的系统搭建概念，并期望跨越手写代码的带宽限制，这套方法论都将为你提供全新的系统级全局视角。

许多人接触 AI 编程的起点是期望寻找“一句话生成完整 App”的银弹。然而，生产级项目充满了复杂的业务边界与长期的迭代需求，缺乏物理边界约束和检验卡口的“魔法生成”，交出来的往往是外表鲜艳、实则一跑就崩的代码。要让交付结果在工程上可靠，我们必须建立起完备的研发生命周期与风控拦截网，而非寄希望于省略开发流程的捷径。

当 Agent 产出的代码出现逻辑偏差时，开发者往往会试图通过编写越来越聪明、复杂的提示词来叮嘱和纠正它。实际上，Agent 的交付可靠性并不取决于更聪明的提示词，而取决于你为它搭建的研发脚手架（Harness）——因为口头提醒极易在长上下文中产生记忆漂移，而 Lint 静态扫描、强类型守卫与自动化卡口等刚性约束才是不可绕过的栅栏。

对于零星的小修小补或几行脚本的编写，直接对话和代码补全确实是最高效的。但这套方法论专注于驱动中大型、需要长期迭代与运维的生产级项目，前期在 Harness（如 Lint、测试卡口与设计系统）上的系统投入，将在未来的数千次提交中被摊薄，从而产生长期的研发复利。本书的核心目的，便是帮助大家跨越这一心智门槛，学会建立机器可强制执行的工程硬约束。

AI 驱动开发是一次研发角色的根本转变。你将从直接编写代码的“作者”，升级为设计约束、建立反馈与评审产出的“环境设计者”。这需要你练习两项新技能：将业务与技术指标翻译为机器可强制执行的硬约束，并建立起高效率的方案与工件评审直觉。这能让你从手写代码的泥潭中抽身，实现研发效能的规模化扩展。

你将掌握如何让 AI Agent 可靠地接管从零立项、规划、开发、测试到灰度发布、运维复盘的完整研发生命周期。通过搭建一套能够限制并辅佐 Agent 自主工作的研发环境，在释放吞吐量的同时，牢牢掌控资金安全与系统质量的底线。

本方法论提炼自少数真实上线、且持续迭代运维的生产项目，非玩具 Demo。它需要在您具体的业务场景中因地制宜地进行取舍，请带着思辨的实战心态阅读，与我们一起在工程实践中摸索进化。

## 如何阅读

为了获得最佳的学习效果，建议循序渐进地按章节顺序阅读。全书在结构上进行了清晰的解耦：第一部分聚焦于纯粹的理论与心智模型，完全不绑定任何具体工具；从第二部分起则逐步进入实操落地。

每章均分为两层进行讲解——前半部分阐述可迁移的通用架构原则，后半部分则展示基于 [gstack](https://github.com/garrytan/gstack) 的参考实现。即便在不同的技术栈中，这些原则依然可以无缝迁移，只需对具体实现进行平替。

## 全书母题：Harness = 文档 + 检查 + 反馈

AI Agent 的研发可靠性，终归来自于为其量身定制的研发脚手架（Harness），而非不断精细化的提示词工程。

在这套体系中，**文档**是将项目宪法红线与历史设计决策记录为物理文件，作为 Agent 每次冷启动开工时无可规避的输入，从而约束输入边界。

**检查**充当主干合并前的自动化拦截卡口，将 Lint 静态扫描、编译约束、单元测试及架构依赖规则强行固化为流水线门禁，保障输出的安全性，拦截输出偏差。

**反馈**则涵盖本地报错与线上结构化可观测性，其核心要求是必须赋予 Agent 最小受控权限的日志抓取权限，并强制要求其在无头浏览器中运行端到端的业务自验证，驱动闭环自愈。这三者共同构成一套自愈的负反馈控制回路，而这一回路在时间线上的展开，即是 `Think` → `Plan` → `Build` → `Review` → `Test` → `Ship` → `Reflect` 七阶段研发生命周期。

## 示例项目：Relay

Relay 是全书贯穿始终的示例项目，它是一个中性虚构的 AI 客户支持自动化 SaaS。系统集成了多渠道会话接入，利用大语言模型（LLM）实现分类与路由、起草回复，并能自主调用外部工具（如检索知识库、查询订单、发起退款或赠送积分）。为了确保交易安全，Relay 设计了分级风控机制：低风险会话由 AI 自动答复并执行，高风险操作则安全挂起并转由人工坐席审批，计费模式则采用按已解决对话收费。

作为一个典型的 Agentic 产品（结合 LLM、工具链与有副作用的实体操作），Relay 具备真实项目运行时的所有复杂特征。全书的工程实践都将围绕它展开，通过真实的配置文件、报错日志与调试迹线，为您提供具备工程摩擦力的实战参考。
